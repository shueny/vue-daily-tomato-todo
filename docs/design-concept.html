1行事曆:藏在標題日期後面

不新增按鈕 —— 直接讓現有的大日期「06 Jul」變成入口,點一下往下展開整月。這是使用者最直覺會去點的地方。

  • 每格下方的小圓點是狀態摘要:灰點=有任務、紅點=有未完成/逾期、綠點=全部完成
  • 每一天都可以點 → 行事曆收合、卡片滑到那一天;沒任務的日子只是顯示淡一點,點過去就能直接新增
  • 今天永遠有白色外框,目前檢視日反白

2建立任務時選日期:預設跟著你看的那天

輸入框右側加一顆日期 chip。核心規則:chip 預設 = 目前檢視中的卡片日,所以在「明天」的卡片上打字直接就是建明天的任務,大多數情況不用多點一下。

  • 點 chip → 快速選單(今天/明天/下週一)+ 完整日期選擇器
  • 資料上就是 todo 多一個 date: 'YYYY-MM-DD' 欄位,和現有 dueDate 合併為同一概念

3每日卡片左右滑動

白色清單區改成水平卡片軌道,一張卡=一天。手機用手勢滑、桌機可用膠囊/行事曆點選。

  • 實作用 CSS scroll-snap,零依賴、原生慣性手感,天數可無限往前後生成
  • 上方「昨天|今天|明天」膠囊跟著捲動位置亮起;離開今天時出現「回今天 ↩」
  • 昨天沒做完的事,今天卡片頂端出現紅色提示列,一鍵「移到今天」(見今天卡片)
  • 篩選(All/Complete/Incomplete)作用在目前卡片

🍅蕃茄鐘:開始即全畫面接管

按任務列的 ▶ 之後,整個畫面被專注遮罩蓋住 —— 只留倒數時間、任務名稱、⏸ 暫停/▶ 繼續、■ 停止,其他一切(清單、行事曆、輸入框)全部看不到。在 mockup 裡按任一 ▶ 試試。

  • 刻意讓你無事可做:專注期間不能滑清單、不能加任務,防分心是功能不是限制
  • 外圈黃色進度環隨秒數消耗,一眼看出這顆蕃茄吃掉多少
  • 半透明灰遮罩 + 背景模糊 —— 清單隱約可見但不可操作,大字倒數與黃色進度環浮在上面
  • ⏸ 暫停會停秒數並顯示「已暫停」;■ 停止放棄這顆蕃茄回到清單
  • 實作:pomodoro store 加 paused 狀態;專注結束自動切換休息畫面(綠色調),休息完回清單

需要你拍板的三個問題

  1. 沒選日期的任務要歸到「今天」,還是另外給一張「未排程」卡片?(建議:歸今天,簡單)
  2. 過去未完成的任務要自動滾到今天,還是留在原日+紅色提示手動移?(mockup 採手動移,保留「昨天沒做完」的誠實記錄)
  3. 既有的 dueDate 欄位與新的 date 合併成一個「這天要做」的概念,可以嗎?
